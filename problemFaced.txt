1."While developing a MERN e-commerce application, my MongoDB Atlas connection kept failing with querySrv ECONNREFUSED. I verified that the database, credentials, and code were correct by testing with MongoDB Compass and nslookup. I narrowed the issue down to Node.js's DNS resolution of SRV records. By testing with the Node dns module and temporarily switching to Google's public DNS using dns.setServers(['8.8.8.8']), I confirmed the root cause was DNS resolution rather than MongoDB
 or my application code. This taught me to debug systematically by isolating each layer—application, driver, DNS, and network—instead of assuming the code was at fault."
